Features

Main Features of Cerberus

Cerberus, our flagship product is built to serve Saas applications of any size and complexity, whether single, or multi-tenant. As a drop-in, it seamlessly operates alongside any other frameworks you might be using. You can choose to use our hosted service, or host it on-premises yourself.

Fine-grained access control

About Us

Blazingly fast, robust and secure

Our aim is to enable SaaS app developers to leverage a powerful Authorization as a Service (AaaS) with minimal effort, empowering end-users to manage their own permissions with clear and easy-to-understand rules.

Cerberus gives you client libraries and UI components out of the box that makes it a breeze to integrate and for your customers to get the benefit of flexible permissions.

Any Questions? Answered

We'll try to cover some common questions.

Documentation and examples will soon become available. Here's one example app that you can refer to in the meantime. Example App
It's a spin-off from an earlier implementation that I did for a company a number of years ago: B2B SaaS Authorization The implementation has changed quite a bit, matured, and turned into an AaaS app itself.
Needless to say, we take security extremely serious, and any data you store on our cloud servers are protected using modern methodologies. That said, we really don't store any sensitive data, except for perhaps user names. Mostly, only database id's of your resources are stored. And if that's still an issue, you can always host Cerberus on your own premises easily using Docker.
I'm working at full speed creating documentation and videos. Hopefully in Q1 2023 a commercial cloud offering will become available, and perhaps sooner a licensed docker installation.
At the moment, I've only developed React components and a Go client library. More libraries and UI components will become available in due course. Feel free to browse the opensource libraries to see how they work: a11n-io github
Honestly, it's a piece of cake. With very minimal code (~20 lines), you can create a fully secure app that gives your users a ton of useful functionality to manage their own permissions, roles, etc.
